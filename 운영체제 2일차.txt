운영체게 3가지 핵심개념
1 가상화
- 물리적인 자원을 이요해서 cpu가 1개인데 다중의 사용자가 독점적으로 시스템을 사용하는 것과 같은 환경을 제공하는것

- cpu가상화(process, Thread, CPU Scheduling)
- 메모리 가상화, 메모리 제한이 없는것처럼 사용하는것
- 가상머신, 컴퓨터시스템 운영체제가 다른 운영체제가 돌아가는것 처럼 제공하는것

2 병행성
전부 잘 이해해야함. 어떤 개발언어를 사용하던지 병행성은 발생한다
- 다중의 프로세스가 동시에 실행되는것을 병행처리라고 말한다.
두가지 방법이있는데
1 한개의 cpu가 번갈아가면서 실행되는것
2 여러개의 cpu일때 병렬적으로 실행되는것
병행처리는 공유자원을 접근할때 문제가 생길수 있음
이 공유자원문제를 해결할수 있어야만 완벽한 프로그램을 짤 수 있음

5장 상호배제와 동기화
6장 교착상태와 기아상태

3 영속성
메인 메모리는 휘발성 메모리이다.
하드디스크와 플래시메모리는 휘발성이 아니다.
운영체제는 파일시스템을 통해 영속성을 지원한다.

여기서 다루지 않는다.
UNIX 시스템 프로그래밍 교과에서 다룬다.
실제 코드를 배운다. 듣기를 권장한다.


운영체제 강의내용
1~9장까지 강의 10장이후로는 다루지 않는다


운영체제란?
*운영체제에선 더블 링크드 리스트를 사용함

UNIX  - CLI(커맨드라인인터페이스) 
최초의 highlevel language를 가진 운영체제
3버전부터 C언어로 작성됨

버클리에서 BSD버전을 만듦
4BSD버전에서 socket이 만들어짐

카네기멜론대
MACH를 제안 - 마이크로 커널구조
기존엔 모든기능을 가진 모놀리틱 커널구조 였음
이 마이크로 커널구조를 확장한게 macOS

리눅스 - 안드로이드의 기본 커널


유닉스 변종이 너무 많이 나와있어서
유닉스 표준이 생김 -> IEEE POSIX
인터페이스로 정의

인터페이스 는 3가지

system call os가 직접 제공
library 운영체제는 아니지만 그 위에서 공통적으로 사용할수있는 함수를 모아둔거임
command 유닉스 명령어

위 3가지를 표준화한것

현재의 표준은 Single UNIX Specification Version 4
C헤더파일, 쉘, 유틸리티, 시스템콜 인터페이스를 포함한다.

리눅스는 저작권료를 안줘도 된다
모든 소프트웨어는 오픈되어야한다는 철학에 힘입어 많은도움을 받음
안드로이드 커널로도 사용했다.
