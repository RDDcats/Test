

공인주소 - 라우터가 알고있는 주소
사설주소
 - 로컬에서 씀 192.168로 시작하는거 10.40으로 시작하는거
 - 라우터들이 모르는 주소
 - 게이트웨이를 타고 밖으로 못나감 > 주소변환을 해야함

IPS- 침입방지 시스템(쓰레기 막아줌)
게이트웨이 - 바깥으로 정보를 보낼수있음, 라우터로 보냄
라우터는 주소가 어디 라우터에 달렸는지 알고있음

라우터는 클래스 기반으로 안한다
클래스 기반 - 정부 학교등 등록된거 확인 > 서브넷확인 > 디폴트
classless- 테이블에 없으면 버린다
라우터가 테이블을 생성해서 엔트리를 관리

참고
{
"Classful하게 찾는 다는 말은 어떤 패킷이 라우터로 들어왔을때
major network단위의 라우팅 테이블을 lookup하고 매치가 안된다면 subnet
마스크를 참조하며, 그래도 매치가 안되면 당연히 default route를 찾아야 하지만, 찾지 않고 packet을 drop하는 것을 이야기 하는 겁니다."

"classless하게 찾는 다는 것은 일단 specific한 subnet까지 고려하여
라우팅 테이블을 뒤져보고 그래도 없으면-major단위로 뒤져보겠져.longest prefix match라고 합니다.-그런데도 없다면, default route를 이용하는 것을
이야기 합니다."
}

포트번호
ip주소가있으면 컴퓨터까지는 감
패킷이 들어오면 버퍼에 두다가 버퍼가 넘어설수 있음
그럼 버려짐(데이터 손실)
그걸 tcp가 해준다
송신자를 구별하기위해 사용한다

전송할때
목적지 ip주소 목적지 포트번호 출발지 ip주소 출발지 포트번호를 키로 쓴다

하나의 프로그램이 여러 소켓을 만들수있다.
포트번호가 다 똑같음
클라이언트는 소켓하나만쓸때
서버에서 리스닝소켓을 띄움
클라가 거기다 연결요청을 함
똑같은 서버소켓을 복사해서 요청을 받음
클라가 늘어날때마다 리스닝 소켓이 복사됨
근데 복사라 포트번호가 똑같음

그래서 tcp는 4개의 키를 쓴다
6만 5천개까지 만들수 있다
